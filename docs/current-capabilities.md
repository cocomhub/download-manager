当前功能特性总览

## 产品定位

- 面向多源任务的下载管理器，覆盖采集、排队、下载、状态追踪、聚合浏览与基础配置治理
- 既可运行完整下载服务，也支持仅启动管理界面的 UI 模式，适合本机或内网环境下的轻量使用

## 当前能力边界

- 运行模式
  - full：启动任务、调度、下载与 Web 管理接口
  - ui：仅提供管理界面与只读浏览场景
- 任务类型
  - url_list：通用 URL 列表下载
  - tktube：分页抓取、详情解析、内容分组、优先级淘汰
  - hanime：分页抓取、Cookie 请求头、HLS 下载链路
  - vikacg：页面/用户帖子抓取、图片资源组织、增量刷新
- 下载执行
  - 支持原生 HTTP 下载与 wget 下载器切换
  - 支持域名限流、进度回调、取消、重试、复合文件下载
  - 支持 m3u8/HLS 与 ffmpeg 处理链路
- 状态管理
  - 支持对象 pending/downloading/completed/failed/canceled 生命周期
  - 支持对象取消、撤销取消、批量取消、失败重试、对象重排
  - 支持基于 URL 的共享状态对齐
- 数据持久化
  - 支持 memory、file、mongo 三种存储后端
  - 文件存储支持延迟落盘与显式 flush
- 管理与浏览
  - 支持任务列表、活动下载、聚合视图、分页、搜索、筛选、排序
  - 支持内容分组视图与组内对象展开
  - 支持 SSE 实时事件推送
- 配置治理
  - 支持配置默认值、兼容迁移、在线更新、备份、回滚、diff、标签与备注
  - 支持运行时调整全局 worker、任务并发、刷新间隔

## 核心模块映射

- 启动与运行模式
  - `main.go`：参数解析、配置加载、单实例锁、Manager 与 HTTP 服务启动
- 配置模型
  - `config/`：配置结构、默认值、兼容旧字段、运行模式与下载器配置
- 任务抽象与工厂
  - `core/interfaces.go`：Task、Storage、Downloader 核心接口
  - `task/factory.go`：任务注册、能力注入、实例化
  - `task/base_task.go`：通用状态回填、关闭流程与能力组合
- 下载编排
  - `manager/manager.go`：扫描、入队、调度、worker、对象控制、聚合查询、事件广播
  - `manager/runtime_mgr.go`：运行时并发与刷新间隔更新
  - `manager/task_loader.go`：任务与存储装载、共享注册表与下载器注入
- 下载执行
  - `downloader/`：原生下载器、wget 下载器、抓取辅助
  - `pkg/dlcore/`：HTTP 下载、HLS/ffmpeg、文件系统封装
- 存储
  - `storage/`：memory/file/mongo 三类实现
- Web 与 API
  - `api/server.go`：REST API、SSE、文件访问
  - `web/static/index.html`：内嵌前端页面
  - `web/embed.go`：静态资源嵌入

## 用户可见功能

- 任务管理
  - 创建、更新、查看、取消、重试任务
  - 查看任务下载对象与当前运行状态
- 对象管理
  - 查看对象状态、下载路径、附加信息与失败情况
  - 手动取消、恢复、重试、重排对象
- 聚合浏览
  - 跨任务统一查看对象
  - 按状态、类型、搜索词、排序方式过滤
  - 按内容组聚合并查看组内对象
- 配置运维
  - 查看与修改服务配置
  - 查看配置历史、差异、标签与备注
  - 进行配置回滚与运行时参数调整

## 当前优势

- 主干闭环完整：采集、下载、持久化、UI、配置治理均已具备
- 扩展基础已具备：任务工厂、通用分页器、刷新器、路径策略可复用
- 实用性较强：支持 UI 模式、SSE 推送、配置回滚、内容分组等较成熟能力

## 当前约束

- `manager` 职责较重，编排、聚合、对象操作、配置治理耦合明显
- 聚合与查询逻辑偏向先全量拉取再处理，大数据量下扩展性有限
- 前端集中在单个大型 HTML 文件中，后续迭代与测试成本较高
- 安全边界偏弱，更适合本机或受控内网环境使用
