# Download Manager 现状能力与优化方向 Spec

## Why
当前项目已经具备较完整的多任务下载、聚合管理、配置回滚与内嵌 Web UI 能力，但缺少一份面向“当前能力现状 + 下一步优化方向”的统一规格文档。
补齐该规格后，可以帮助后续迭代以一致视角识别已有能力、明确短板，并把优化事项转化为可执行的实现任务。

## What Changes
- 新增一份“现状能力与优化方向”规格，沉淀项目当前功能边界、关键模块与用户可见能力
- 新增一组面向后续实现的优化主题，包括写操作保护、文件访问路径一致性、停机流程、聚合查询性能、配置面完整性、前端可维护性与安全加固
- 为优化主题提供后续实施任务拆分与验收清单，作为下一阶段改造入口

## Impact
- Affected specs: 当前能力盘点、优化路线规划、服务端治理、前端维护性、部署安全性
- Affected code: main.go、manager/、api/server.go、config/、storage/、task/、web/static/index.html

## ADDED Requirements
### Requirement: 当前功能特性规格化
系统 SHALL 提供一份可追踪的能力规格，覆盖当前项目已经支持的核心功能、关键模块职责与主要用户入口。

#### Scenario: 梳理当前核心能力
- **WHEN** 维护者审阅该规格
- **THEN** 可以明确识别项目已支持的任务类型、下载调度、状态管理、聚合视图、配置管理、存储后端与内嵌 Web UI 能力

#### Scenario: 梳理系统边界
- **WHEN** 维护者使用该规格作为后续改造输入
- **THEN** 可以从规格中定位受影响的主要模块，而不需要先反复通读整个代码库

### Requirement: 优化方向规格化
系统 SHALL 提供一份与当前实现对应的优化方向清单，覆盖已识别的高价值改进点，并说明其目标与影响面。

#### Scenario: 明确高优先级优化主题
- **WHEN** 维护者规划下一轮迭代
- **THEN** 可以从规格中看到至少以下优化主题：写操作保护一致性、文件访问路径一致性、优雅停机、聚合/查询性能、配置接口补全、前端模块化、安全加固

#### Scenario: 将优化方向转化为实施任务
- **WHEN** 团队准备落地某一优化主题
- **THEN** 可以直接参考该变更下的 tasks.md 与 checklist.md 进行开发和验收

### Requirement: 现状与优化并存的路线图表达
系统 SHALL 在同一份规格中同时表达“已经具备的能力”与“建议补齐的能力”，以避免把现状分析与改造规划割裂。

#### Scenario: 同时查看现状与改造建议
- **WHEN** 维护者查阅该规格
- **THEN** 能同时理解当前项目的完成度、主要风险点以及下一步建议动作

## REMOVED Requirements
